## ADR-001: フレームワークに Next.js 14 (App Router) を採用

**日付**: 2026-06-12  
**状態**: 採用

**背景**: フロントエンドとAPIを同一リポジトリで管理したい。SSRによるSEO最適化も必要。

**決定**: Next.js 14 App Router を採用。  
**理由**: API Routes でバックエンドロジックを完結でき、Server Components によるパフォーマンス最適化が可能。Vercelへのデプロイが容易。

## ADR-002: 月柱の計算基準を節気（節入り）とする

**日付**: 2026-06-12  
**状態**: 採用

**背景**: 四柱推命の月柱計算には「旧暦月」基準と「節気（節入り）」基準の2流派がある。

**決定**: 節気（節入り）基準を採用。  
**理由**: 正統的な四柱推命では節気基準が主流。lunar-javascript ライブラリが節気データを提供しており実装が容易。監修鍼灸師の確認済み。

## ADR-003: AI アドバイス生成に Claude API を使用

**日付**: 2026-06-12  
**状態**: 採用

**背景**: 命式に基づく養生アドバイスの自動生成が必要。

**決定**: `claude-sonnet-4-6` モデルを使用。  
**理由**: 日本語の自然な出力品質が高く、東洋医学の文脈理解が優秀。コスト面でもOpusより現実的。

## ADR-004: データベースに Supabase を採用

**日付**: 2026-06-12  
**状態**: 採用

**背景**: ユーザーの診断結果を保存・再表示する機能が必要。

**決定**: Supabase (PostgreSQL) を採用。  
**理由**: Next.js との相性が良く、認証機能が組み込まれている。個人開発規模では無料枠で運用可能。

## ADR-005: 体質分類を中医学九体質に準拠

**日付**: 2026-06-12  
**状態**: 採用

**背景**: 五行分析の結果をユーザーに分かりやすく伝える分類体系が必要。

**決定**: 中医学の九体質（気虚・血虚・陰虚・陽虚・気滞・血瘀・痰湿・湿熱・平和質）に準拠。  
**理由**: 国際的に認められた中医学の標準的体質分類。鍼灸師監修のもと正確な説明が可能。
